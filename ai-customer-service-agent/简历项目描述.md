# AI 客服工单处理 Agent（开源项目）

**2026.3–2026.5**

**项目背景**：面向 AI Native/Agent 岗位求职，从零构建一个技术完整度可对标企业级产品的 AI Agent 开源项目。旨在展示对 LLM 应用架构、Agent 工作流编排、RAG 检索增强、Tool Calling 及前后端全栈交付的工程能力。

**项目执行**：

1. **Agent 工作流编排与 Tool Calling**：设计并实现从工单创建→LLM 分析→RAG 检索→工具调用→流式回复→工单总结的完整 Agent 闭环链路。定义 4 个 OpenAI 兼容的 Tool Definition（查订单、退款政策、产品手册、升级人工），Agent 根据分析结果自主决策调用工具，查询结果注入回复生成的上下文窗口。

2. **Hybrid RAG 检索系统**：构建关键词评分检索 + 向量语义检索的双路融合架构。关键词通道对标题（权重 3）、关键词字段（权重 2）、正文词频（权重 0.5）进行加权评分；语义通道通过 DeepSeek Embedding API 向量化后计算余弦相似度；两路结果去重合并，解决单路检索的覆盖盲区。支持 RAG_MODE 配置在 hybrid/embedding/keyword 间动态切换。

3. **LLM 分析与 SSE 流式回复**：基于 OpenAI 兼容协议接入 DeepSeek API，通过 JSON Mode 实现工单结构化分析（情绪识别/问题分类/优先级判断/人工升级建议）。采用 SSE（Server-Sent Events）协议实现流式回复，后端 sse-starlette + 线程队列异步生成，前端 fetch ReadableStream 逐帧解析渲染，实现打字机效果。

4. **降级策略与零依赖设计**：实现 LLM→规则引擎的自动降级：LLM 不可用时自动切换关键词规则引擎（情绪/分类/优先级关键词匹配 + 模板回复），确保项目零外部依赖仍可完整运行。分析、检索、回复生成、总结四个关键环节均有 fallback 机制。

5. **全栈工程实践**：后端 FastAPI + SQLAlchemy + SQLite（21 个 REST API + SSE 端点），前端 Next.js 16 + React 19 + TypeScript + Tailwind CSS 4（6 个页面，含流式 Agent 工作台）。懒加载单例模式管理 LLM 客户端和 Embedding 服务，减少重复初始化开销。

**项目结果**：GitHub 开源，覆盖工单管理、AI 分析、RAG 检索、Tool Calling、流式回复、工单总结、运营统计 7 大核心功能。支持 DeepSeek/OpenAI 等任意 OpenAI 兼容 API，配置即用。前端 6 页完整交互界面，后端 21 个 API 端点，29 个冒烟测试场景全部通过。
